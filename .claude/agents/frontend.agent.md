# Frontend Execution Core

## Mission

Build pages that are compact, readable, fast to scan, and task-first on mobile and desktop.
Do not ship generic scaffold layouts.

## Hard Rules

1. Binance is the structure benchmark: density, hierarchy, and action discoverability.
2. Every UI task must pass 320, 768, 1024, 1440 checks.
3. Every UI task must emit route-level screenshot artifacts and scorecards.
4. No tool claims beyond frontmatter tools.
5. If evidence is missing, task status is INCOMPLETE.

## Design System Contract

### Tokens

- Spacing: 4, 8, 12, 16, 20, 24, 32, 40, 48.
- Radius: 8, 12, 16.
- Border width: 1.
- Shadow level-1: 0 1px 2px rgba(0,0,0,0.08).
- Shadow level-2: 0 4px 12px rgba(0,0,0,0.10).

### Typography

- Mobile
  - h1: 28/34 700
  - h2: 24/30 700
  - h3: 20/26 600
  - body: 14/20 400
  - caption: 12/16 400
- Desktop
  - h1: 36/44 700
  - h2: 30/38 700
  - h3: 24/32 600
  - body: 15/22 400
  - caption: 12/18 400

### Grid Math

- 320: 4 columns, margin 16, gutter 12.
- 768: 8 columns, margin 24, gutter 16.
- 1024: 12 columns, margin 32, gutter 20.
- 1440: 12 columns, max width 1280, gutter 24, centered frame.

### Density Tiers

- Compact (default marketplace/dashboard): row 56, card padding 12-16, section gap 16.
- Comfortable (forms/details): row 64, card padding 16-20, section gap 20.

## Component Anatomy

1. Top bar
   - Mobile 56, desktop 64.
   - Sticky when page has filters or search.
2. Utility/filter rail
   - Chip height 32.
   - Horizontal scroll on mobile.
3. Primary card
   - Header (title + state), metric strip, action row.
   - Mobile stack, desktop split layout.
4. Data list/table
   - 320/768: card-list mode.
   - 1024/1440: table allowed when readable.

## Required Workflow

1. Identify route and user path.
2. Map files in scope: page, layout shell, theme, shared components.
3. Implement with tokens and grid contract.
4. Run auto-remediation for safe spacing/overflow fixes:
   - npm run ui:auto-remediate -- --changed --apply
5. Run route pack compare:
   - npm run ui:pack:compare -- --pack core-public --task-id <task-id> --strict true
6. If fail, iterate until pass.

## Required Evidence

Per route task, required artifacts:

- .artifacts/ui/<task-id>/320.png
- .artifacts/ui/<task-id>/768.png
- .artifacts/ui/<task-id>/1024.png
- .artifacts/ui/<task-id>/1440.png
- .artifacts/ui/<task-id>/scorecard.json
- .artifacts/ui/<task-id>/issues.json

Scorecard pass rule:

- visualHierarchyScore
- spacingScore
- positioningScore
- interactionScore
- densityScore
- total >= 22/25 and no category < 4

## Failing Conditions

Mark task FAIL if any of these occur:

1. Horizontal overflow or clipped critical controls.
2. Mobile tap targets below 44px for primary interactions.
3. Visual diff mismatch above threshold.
4. Missing artifacts.
5. Scorecard below threshold.

## Anti-Targets

1. Even-weight boxes without hierarchy.
2. Tall hero sections that hide utility controls.
3. Random spacing values outside token scale.
4. Layouts that rely on whitespace instead of structure.
5. Desktop pages that are wider but not more structured.

## Tool Reality

Use only declared tools.
If visual tooling is missing, create and run command-based checks before claiming success.
